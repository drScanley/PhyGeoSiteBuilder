# PhyGeo — Ancestral Range Explorer

An interactive web figure for exploring the output of [PhyGeo](https://github.com/js-arias/phygeo) ancestral-range
reconstruction across a phylogeny (In this example that of girdled and plated
lizards of Africa and Madagascar). The model is described in [Arias (2025)](https://doi.org/10.1093/sysbio/syae051).
Instead of flipping between dozens of static map images, viewers hover or tap 
directly on the nodes of the phylogenetic tree and the corresponding range map 
— the posteriorprobability surface for where that lineage lived — appears alongside it.

## What this shows

Each node in the tree represents either a present-day species (a tip) or
an inferred common ancestor (an internal node). For each one, a Bayesian
biogeographic analysis estimates the probability that the lineage
occupied a given part of the world at that point in time — visualized as
a heatmap over a world basemap (warm colors = higher posterior
probability). A few key nodes:

- **Root** (~102 Mya) — the deepest node in the tree
- **Node 3** — a fossil taxon (*Konkasaurus mahalana*, ~66 Mya), used as
  a calibration point rather than a living tip
- **Nodes 1 and 30** — the most recent common ancestors of the two major
  families in the group, Gerrhosauridae and Cordylidae
- **Nodes 1, 4, and 5** — additionally include a *branch time-series*: a
  slider that scrubs through several reconstructed time-slices along that
  branch, rather than a single fixed snapshot

## How it works

This isn't hand-built HTML — it's generated by a small Python script
(`build_site.py`) from four source inputs:

1. **`tree.svg`** — the cladogram (e.g. exported from R's
   `ape::plot.phylo`). The script parses the node circles directly out of
   this file to get each node's pixel position, so it keeps working even
   if the tree's shape or size changes.
2. **`manifest.csv`** — a plain-text table mapping each node number to
   its age, its kind (root / internal / tip / fossil / family), a label,
   and which image file goes with it. Giving one node multiple rows (at
   different ages) is what creates a branch time-series slider for it.
3. **`images/`** — the range-map PNGs referenced in the manifest.
4. **`site_config.json`** — the editable header text (title, tagline,
   description) shown at the top of the page.

The script stitches all of this into one self-contained HTML file — the
tree SVG inline, every map image embedded as base64 — so the final output
has no external dependencies and can be opened directly in a browser or
hosted anywhere as a static file.

```
manifest.csv + images/ + tree.svg + site_config.json  →  build_site.py  →  output/site.html
```

## Folder layout

```
site_builder/
  build_site.py     the script — run this
  template.html      page shell (edit CSS if you want to restyle)
  site_config.json     header text: eyebrow, title, description, tab title
  tree.svg            the cladogram (from R's ape::plot.phylo or similar)
  manifest.csv          node, age, kind, label, filename
  images/                 all PNGs referenced in manifest.csv
```

## Usage

```
python3 build_site.py
```

Writes `output/site.html` — a single self-contained file (~2.4 MB), ready
to open directly or host anywhere (GitHub Pages, etc).

No dependencies beyond the Python standard library — no `pip install`
needed.

### Options

```
python3 build_site.py --tree tree.svg --manifest manifest.csv \
                       --images images --template template.html \
                       --config site_config.json --out output/site.html
```

All paths default to the layout above, so you only need flags if you're
deviating from it.

## Customizing the header text

Edit `site_config.json` — plain text, no HTML needed:

```json
{
  "page_title": "Cordylid Biogeography — Ancestral Range Explorer",
  "eyebrow": "Ancestral Range Reconstruction",
  "title": "Cordylid Lizard Biogeography",
  "description": "A Bayesian phylogeny of the Cordylidae (girdled and plated lizards of Africa and Madagascar)."
}
```

Re-run the script and the new text appears — `page_title` is the browser
tab title, `eyebrow` is the small label above the headline, `title` is
the big headline, and `description` is the paragraph underneath it.
If this file is missing or a key is left out, generic placeholder text
is used instead of the script failing.

## Updating with new maps

1. Add the new PNGs to `images/`.
2. Append rows to `manifest.csv` (see the column descriptions in the
   docstring at the top of `build_site.py`). Giving the same `node`
   number to multiple rows turns it into a branch time-series slider,
   ordered by `age`.
3. Re-run `python3 build_site.py`.
